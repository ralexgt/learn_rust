-> Manual testing will get more and more time consuming as the application grows bigger
-> We want to automate as many tests as we can, which will be run during the CI

# How to test an endpoint
-> The logic of the program lives in `lib`. Even though we do not want to publish a library, it is much more clean to have a separation of concerns. `main.rs` will contain only the minimal code to run the server

-> `tokio::test` is like `tokio::main` but for tests. It wraps the function in an async runtime

-> Add dependencies used for tests or examples in `[dev-dependencies]`. This way they are not included in the final binary
